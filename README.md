# BhuMe Boundary Alignment

A lightweight, explainable pipeline for correcting the positions of digitised official land-plot boundaries using satellite imagery and field edge rasters.

I explored several approaches before settling on a classical cross-correlation method, rejecting deep-learning alternatives because of the limited training data available and the need for interpretable confidence scores that actually track accuracy.

## Approach

The alignment pipeline works as follows for each land plot:

1. **Area Validation**: Compares the drawn polygon area with the legal 7/12 land record area. Plots with a severe area mismatch (drawn area less than 50% or greater than 200% of the recorded area) are flagged immediately and left at their official position. This prevents the algorithm from confidently "correcting" plots whose geometry is fundamentally inconsistent with the land register.

2. **Patch Extraction**: Clips the `boundaries.tif` raster around the plot with an 80-metre padding to provide enough context for the search.

3. **Template Rasterisation**: Converts the official boundary polygon outline into a thin template mask representing the plot's edge.

4. **Edge Detection**: Applies a Sobel filter to the raster patch to detect field boundaries in the imagery.

5. **Cross-Correlation**: Slides the template mask over the edge image using FFT convolution to find the translation offset (dx, dy) that maximises overlap between the official boundary and the visible field edges.

6. **Confidence Estimation**: The correlation response provides a natural uncertainty signal. A sharp, dominant peak indicates a unique and convincing alignment candidate, while a flat or ambiguous response suggests multiple plausible placements. The ratio of the peak value to the mean response is used to produce a 0–1 confidence score. Plots with a flat response are either assigned lower confidence or flagged for manual review. This gives the confidence score genuine meaning rather than being arbitrarily assigned.

7. **Shift Reprojection**: Converts the UTM shift back to WGS84 and applies the transformation to the geometry.

### Adaptive Search Windows

Search windows are adapted using plot-size statistics (median area across the village) to avoid implausible corrections in dense settlements while permitting broader exploration in open agricultural areas. For villages where many individual plots aligned confidently, a global consensus offset is computed from the top-confidence shifts and used as a fallback for plots that were too noisy to align individually.

## Evaluation

The included predictions were evaluated using the starter-kit scoring utilities and the BhuMe test page. Metrics reported reflect the outputs generated by this repository.

### Vadnerbhairav
- Median IoU: **0.852** (Official Start: 0.612)
- Centroid Error: 9.9 m
- Accurate Plots (IoU >= 0.5): 66.7%
- Calibration AUC: **0.875**

### Malatavadi
- Median IoU: **0.544** (Official Start: 0.510)
- Centroid Error: 4.1 m
- Accurate Plots (IoU >= 0.5): 66.7%
- Calibration AUC: **0.750**

These scores come from the `example_truths.geojson` files (6 plots for Vadnerbhairav, 3 for Malatavadi). They should be treated as indicative rather than definitive given the small sample sizes.

## Limitations

- Dense villages remain challenging. Closely packed fields mean the search window easily picks up a neighbouring plot's boundary instead of the target, producing ambiguous correlation peaks.
- Heavy canopy and built-up areas degrade the boundary signal in `boundaries.tif`, reducing both alignment accuracy and confidence reliability.
- The pipeline assumes the primary error is a rigid placement drift, not shape deformation. Plots where the recorded shape is genuinely wrong (not just displaced) are caught by the area check and flagged, but edge cases may slip through.
- Global consensus fallback in dense villages applies a median shift from high-confidence neighbours, which may not be appropriate for isolated plots with unusual placement errors.

## Setup and Reproduction

Install dependencies with `uv` (recommended):

```bash
uv sync
```

Or with pip:

```bash
pip install geopandas rasterio shapely numpy scipy pillow pyproj
```

Run the alignment pipeline on a village folder:

```bash
python solve.py data/34855_vadnerbhairav_chandavad_nashik
python solve.py data/12429_malatavadi_chandgad_kolhapur
```

The output `predictions.geojson` will be written alongside the input data in the village folder.

**Expected runtime**: approximately 1–2 minutes per village on a standard laptop (no GPU required).
**Memory**: under 1 GB peak.
